G copilot-instructions.md

GitHub Copilot reads this file automatically on every chat session in this workspace. Put your universal rules here so they are always active — no paste needed.

T tasks.json

Runs extract_prompt.py from a VS Code task menu. Pick a prompt ID from a dropdown; output appears in the terminal ready to copy into any AI chat.

K keybindings.json

One keyboard shortcut opens the prompt picker task without leaving your editor. Works project-wide once committed to .vscode/.

A validate-prompts.yml

GitHub Actions runs extract_prompt.py --validate on every push and PR. Broken XML or schema violations block the merge before they reach the team.

How to invoke in VS Code

1 — Run a prompt extraction

Press CtrlShiftP → type Run Task → select Extract Prompt. A dropdown appears with all prompt IDs. Pick one; the full prompt prints in the terminal. Copy and paste it into GitHub Copilot Chat, Claude, or any AI assistant.

2 — Keyboard shortcut

Press CtrlShiftX (configured in keybindings.json) to jump straight to the Extract Prompt task — skipping the command palette entirely.

3 — Reference the XML directly in Copilot Chat

In Copilot Chat, type #file:.prompts/prompt_library.xml to attach the full library as context. Then ask: "Use the cms_bug_fixing prompt for this issue."

4 — copilot-instructions.md (always-on)

The universal pre-task rules (no new files, wait for PROCEED, etc.) live in .github/copilot-instructions.md. Copilot loads them automatically on every session — no action needed.

5 — Validate before committing

Press CtrlShiftV → runs Validate Prompt Library. Catches XML errors locally before CI sees them. Use CtrlShiftL to list all IDs without opening the XML.

What GitHub CI does automatically

1

Triggered on every push & PR

The workflow runs whenever any file in .prompts/ changes, plus on every PR targeting main. No manual step needed.

2

Installs lxml and validates XML vs XSD

Runs python extract_prompt.py --validate. Fails the job if the XML is malformed, has duplicate IDs, or violates the schema.

3

Smoke-tests every prompt ID

Runs --list then extracts a sample prompt to confirm the parser returns real content, not empty strings.

4

Blocks merge on failure

Set the workflow as a required status check in your branch protection rules. No broken prompt library can reach main.

5

Caches pip dependencies

Uses actions/cache on the lxml install so the job runs in roughly 10 seconds on repeat pushes.
